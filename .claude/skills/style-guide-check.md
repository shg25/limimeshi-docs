## Style Guide Check Agent Skill

このAgent Skillは、ドキュメントの記述スタイルを統一する。

### 参照ガイド

**マスター**: [governance/docs-style-guide.md](../../governance/docs-style-guide.md)

### チェック項目

#### 1. 句読点

**ルール**：箇条書き・説明文では「。」を使わない

```markdown
# NG
- チェーン店一覧を表示する。
- キャンペーン情報を取得する。

# OK
- チェーン店一覧を表示する
- キャンペーン情報を取得する
```

**例外**：注意書き・免責事項では「。」を使用可

#### 2. Phase表記

**ルール**：スペースなしで記述

```markdown
# NG
Phase 1、Phase 2

# OK
Phase1、Phase2
```

#### 3. コロン

**ルール**：全角コロン「：」を使用

```markdown
# NG
Phase1: 設計

# OK
Phase1：設計
```

#### 4. サービス名表記

**ルール**：
- タイトル・正式表記：「期間限定めし（リミメシ）」
- 文中：「リミメシ」も許容

```markdown
# OK（タイトル）
# 期間限定めし（リミメシ）ガバナンスリポジトリ

# OK（文中）
リミメシの管理画面を開発する
```

#### 5. 太字（強調）

**使用する場面**：
- サービス名の初出
- 技術用語の初出
- 重要なルール・警告

**使用しない場面**：
- 一般的な説明文
- 既出の用語

### 検出時のアクション

スタイル違反を検出した場合：

1. 該当箇所を特定して報告
2. 正しい形式を提案
3. 自動修正が可能な場合は修正案を提示

### 使用タイミング

- Markdownファイル（`.md`）編集後
- 新規ドキュメント作成時
- コミット前のレビュー時
